## Project: MuscoviteHarbor

## Infrastructure Requirements

| Service | Version | Purpose |
|---------|---------|--------|
| PostgreSQL | 15+ | Primary data store |
| NATS | 2.10+ | Event streaming (JetStream) |
| gRPC | — | Service communication |

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DATABASE_URL` | Yes | — | PostgreSQL connection string |
| `NATS_URL` | No | `nats://localhost:4222` | NATS server URL |
| `GRPC_PORT` | No | `50051` | gRPC listen port |
| `GRPC_WORKER_THREADS` | No | `4` | gRPC worker thread count |

## Database Setup

### 1. Create database

```bash
createdb muscovite_harbor
```

### 2. Deploy schemas

```bash
muscdb deploy --profile dev --dir generated/database
```

### Schema deployment order

The following schemas are deployed in dependency order:

1. `vessel_traffic` — VesselTraffic
2. `cargo_operations` — CargoOperations
3. `passenger_terminal` — PassengerTerminal
4. `intermodal_transfer` — IntermodalTransfer
5. `cargo_decomposition` — CargoDecomposition
6. `harbour_control` — HarbourControl

## NATS Stream Setup

# Create JetStream streams for each context

## Starting the Server

```bash
export DATABASE_URL="postgresql://localhost/muscovite_harbor"
export NATS_URL="nats://localhost:4222"
export GRPC_PORT=50051

./muscovite_harbor-server
```
